    RecyclerView
    1. View 이면서 Layout처럼 동작을 하는 도구
    2. 데이터들을 List형태로 표현하기 위한 도구
    3. 혼자서는 할수 있는 일이 아무것도 없다.
    4. 내부에 각 데이터(1개의 item)를 표현할 View를 포함해야 작동을 할 수 있다.
    5. 데이터 개수가 화면을 넘어가면 Scroll을 해서 데이터를 확인할 수 있는 구조
    6. ScrollView와의 차이
        ScrollView는 디자인단계에서 내부에 포함할 데이터가 정해진다.
        RecyclerView는 사용하는 과정에서 생성, 추가되는 데이터들을 표현한다.
        (초기에 데이터가 몇개인지 알수 없는 경우 사용)

    사용방법
    1. 내부에 각 데이터를 표현할 View를 만들어야 한다.
    2. 데이터를 담아서 View에 표현할 VO를 만든다.
    3. 안드로이드에서는 VO하고 View하고 직접적으로 연결할 수 있는 도구가 없다.
        그래서 그 중간에 데이터를 변환하고, 연결을 도와줄 Adapter를 만든다.
        (안드로이드에서 제공하는 클래스를 상속받아서 작성하는 클래스)
    4. 각각의 데이터를 itemView하고 연결할 Holder라는 클래스를 만든다.
        (안드로이드에서 제공하는 클래스를 상속받아서 작성하는 클래스)
    5. Holder를 Adapter에 주입시키지
        Holder를 Adapter에 사용할 수 있도록 인식시키는 절차
